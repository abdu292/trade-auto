1) VOLATILITY & ADR/ATR ENGINE
--------------------------------
Implement functions:

- CalcADR20():
  - Using last 20 daily candles high‑low, compute ADR.
- CurrentDayRange():
  - TodayRange = TodayHigh − TodayLow.
  - ADR% = TodayRange / ADR20 * 100.

- CalcATR(timeframe, period):
  - ATR_M15, ATR_H1.

- MeanDistanceFromMA(timeframe):
  - distance = |price − MA20| / ATR(timeframe).

- PhaseClassifier():
  - Based on ADR%, ATR slope, and price vs MA20:
    - Compression (low ADR%, flat ATR).
    - Early Expansion (ADR rising, ATR rising, price breaking range).
    - Late Expansion–Exhaustion (ADR high, price far from MA20).
    - Early Mean‑Reversion (price snapping back toward MA20).
    - Shock (extreme range, gaps, news tag).

Apply hard laws:
- ADR ≥120 → block all new trades.
- ADR ≥95 → block Rail‑B.
- Mean distance >1.5 ATR for Rail‑B; >2.5 ATR for Rail‑A unless deep reclaim.

2) SESSION, DAY, WEEK FILTERS

4) STRUCTURE & S/R DETECTION
-----------------------------
Within MT5 EA or external code:

- Identify:
  - Daily high/low.
  - Last significant swing highs/lows on M15/M30/H1.
- Build S/R map:
  - S1: nearest significant support below price.
  - S2: deeper support where flushing likely.
  - R1: nearest resistance above price.
  - R2: sweep pocket above R1.
  - FAIL: level that invalidates long bias (e.g., key daily/weekly low).

Implement mid‑zone detection:
- If price between S1 and R1 without clear structure → mid‑zone (no new orders).

5) SPIKE / LEG‑2 / WATERFALL DETECTION
---------------------------------------
Implement pattern detection:

- SpikeScanner():
  - On M5/M15, detect bars where range > X × ATR (e.g., ≥1.5 ATR).
  - Cluster multi‑bar expansions:
    - If 2–3 big bars consecutively with ADR jump → candidate waterfall.
- Classify:
  - Leg‑1: first strong break from compression.
  - Leg‑2: continuation move after a pullback/compression.
  - Exhaustion: expansion with many upper wicks / long tails, followed by sideways/overlap.
  - Waterfall: multiple large red bars down (for long side risk).

Impulse Decay Gate:
- For potential leg‑2 long (Rail‑B):
  - Reject if:
    - ATR slope flat or down after spike.
    - ≥2 overlapping candles near highs.
    - ≥2 long upper wicks.
    - Volume (if available) dropping.

Waterfall Shield 2.0:
- If ≥2 of:
  - 3 wide M15 candles in sequence.
  - >1.3×ATR15 in <20m.
  - >20% ADR consumed in <30m.
  - New negative NEWS geo shock tag.
→ Block all new longs until at least X compression bars (e.g., 8) appear.

6) RAIL SELECTION & ENTRY PLACEMENT
------------------------------------
Rail‑A (BUY LIMIT reclaim/value):
- Candidate levels: S1/S2 or reclaimed swept lows.
- Check:
  - Bias from NEWS/ANALYZE: Rail‑A allowed.
  - Level is not mid‑zone.
  - Segment is Compression / Early Expansion / Early Mean‑Reversion.
- Place BUY LIMIT at level ± buffer:
  - Eg. entry = S1 + small ATR fraction.

Rail‑B (BUY STOP continuation — leg‑2 harvester):
- Candidate: above compression high.
- Conditions:
  - ADR <95.
  - Mean‑distance ≤1.2–1.5×ATR.
  - Compression pre‑break: at least 6 candles.
  - Breakout acceptance: at least 2 body closes above box.
  - Impulse Decay Gate passes.
  - No hazard within 90 minutes of intended ML.
  - Not Thursday US hazard zone; not late session.

7) PE & ML TIME LOGIC
----------------------
PE (Pending Expiry):
- For each pending order, set MT5 expiry field to PE.
- Derive PE from:
  - Setup type:
    - US leg‑2: PE 10–20m.
    - London expansion: 20–40m.
    - M15 reclaim/value: 20–45m.
    - H1 reclaim: 45–90m.
  - Clamp to:
    - Before hazard buffers,
    - Before SOS boundary,
    - Inside current session.

ML (Max‑Life):
- Implement as:
  - A time‑check inside EA:
    - For each open position, store ML timestamp in order comment or global variable.
    - Periodic timer checks current time ≥ ML → close or move to scratch logic.
- ML baselines:
  - Leg‑2: 12–25m.
  - London expansion: 25–45m.
  - M15 reclaim: 30–60m.
  - H1 reclaim: 45–90m.
- Scratch rule:
  - Attempt to close near BE (±2 USD) if TP not hit by ML.
